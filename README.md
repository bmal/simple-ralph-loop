# simple-ralph-loop

`ralph` runs a prompt in a full-auto coding-agent session while refusing
metered LLM API authentication. The initial release supports macOS and
OpenCode authenticated with a ChatGPT subscription or Claude Code authenticated
with a Claude subscription.

Install with `pipx install .`, then run with a finite budget of 1 to 100 fresh
sessions:

```sh
ralph run prompt.md --backend opencode --iterations 5
ralph run prompt.md --backend claude --iterations 5
```

Use `--worktree PATH` to target another GitHub worktree. OpenCode defaults to
`openai/gpt-5.6-sol`; Claude defaults to `claude-opus-4-8`. Use `--model` to
select another model from the chosen subscription-backed provider.

Claude requires a Pro or Max login through `claude.ai`, or a token from that
subscription generated by `claude setup-token` and supplied as
`CLAUDE_CODE_OAUTH_TOKEN`. Team and Enterprise accounts are refused because
server-managed settings cannot be proven isolated. Ralph also refuses
Console/API credentials, cloud providers, gateways, custom endpoints, and
ambiguous routing.

Ralph stops early on an exact completion marker. If the budget is exhausted,
it exits non-zero. Runtime state is private to the selected worktree's Git
directory. Remove only that state when no loop is active with:

```sh
ralph clean --worktree PATH
```

Full-auto mode can edit files and run commands without confirmation. Review
the prompt and repository before starting Ralph.
